Aula Prática 1: Configurando o Ambiente e "Hello, World!"

Objetivo

Garantir que todos os alunos tenham o ambiente de desenvolvimento a funcionar e criem a sua primeira aplicação React Native.

Passos

  1. Instale o Node.js (versão LTS) e o Yarn.
  2. Instale o Expo CLI globalmente com `npm install -g expo-cli`.
  3. Crie um novo projeto com `expo init meuPrimeiroApp`.
  4. Abra o projeto no VS Code e explore a estrutura de pastas.
  5. Inicie o projeto com `yarn start` e abra-o no emulador/simulador.
  6. Instale a app Expo Go no seu telemóvel e teste a aplicação via QR Code.
  7. Modifique o ficheiro `App.js` para exibir "Olá, Mundo! O meu nome é [Seu Nome]".

Aula Prática 2: Componentização e Estado

Objetivo

Passos

  2. Crie um componente `Header.js` que recebe uma `prop` `title` e a exibe.

  5. Adicione dois botões ao `Counter`: "Incrementar" e "Decrementar", que alteram o estado.
  6. Importe e use os componentes `Header` e `Counter` no seu `App.js`.

Aula Prática 5: Implementando a Navegação Inicial

Objetivo

Criar um fluxo de navegação funcional entre a tela de login e a lista de tarefas usando o Stack Navigator.

Passos

  1. Instale as dependências do React Navigation (`@react-navigation/native`, `@react-navigation/stack`, etc.).
  2. Envolva toda a aplicação com o `NavigationContainer` no `App.js`.
  3. Crie um Stack Navigator e adicione `LoginScreen` e `TaskListScreen` como telas.
  4. Modifique o `onPress` do botão de login para chamar `navigation.navigate('TaskList')`.

  6. Exiba uma mensagem de boas-vindas na `TaskListScreen` usando o email recebido.

Aula Prática 7: Hooks Avançados na Prática

Objetivo

Usar `useEffect` para simular uma chamada de API e `useContext` para gerir um estado global, como um tema.

Passos

  1. Na `TaskListScreen`, crie um estado para as tarefas e outro para o carregamento (`loading`).
  2. Use `useEffect` para, após 2 segundos (`setTimeout`), carregar dados mock no estado de tarefas e definir `loading` como `false`.
  3. Exiba um `ActivityIndicator` (spinner) enquanto `loading` for `true`.
  4. Crie uma pasta `src/contexts` e um ficheiro `ThemeContext.js`.
  5. Crie um contexto de tema (com valores para `light` e `dark`) e um Provider que permita alternar entre eles.
  6. Use o `useContext` nas suas telas para aplicar cores de fundo e de texto com base no tema selecionado.

Aula Prática 9: Implementando Login e Cadastro

Objetivo

Substituir a navegação falsa por um sistema de autenticação real com o Firebase Authentication.

Passos

  1. Na `LoginScreen.js`, importe as funções de autenticação do Firebase.
  2. Crie uma função `handleSignUp` que usa os estados de email e senha para criar um novo utilizador.
  3. Crie uma função `handleSignIn` para fazer o login de um utilizador existente.
  4. Adicione um botão de "Cadastrar" e conecte as funções aos botões corretos.
  5. Use `onAuthStateChanged` na sua estrutura de navegação para alternar entre as telas de Login e as telas principais da app.
  6. Adicione um botão de "Sair" que chama a função `signOut`.

Aula Prática 10: Salvando e Exibindo Tarefas do Firestore

Objetivo

Implementar a criação de novas tarefas e a leitura delas em tempo real a partir da base de dados Firestore.

Passos

  1. Na `AddTaskScreen`, crie um input para o título da tarefa e um botão "Salvar".
  2. Ao pressionar "Salvar", use `addDoc` para criar um novo documento na coleção `tasks` no Firestore.
  3. O documento deve conter o título, um status (`completed: false`) e o ID do utilizador logado.
  4. Na `TaskListScreen`, use `useEffect` com `onSnapshot` para ouvir em tempo real as tarefas do utilizador logado.
  5. Atualize o estado `tasks` com os dados recebidos do `onSnapshot`.
  6. Teste: crie uma tarefa e veja-a aparecer instantaneamente na lista.

Aula Prática 11: Completando e Deletando Tarefas

Objetivo

Implementar as funcionalidades de atualização e exclusão (Update, Delete), completando o ciclo CRUD.

Passos

  1. Em cada item da lista de tarefas, adicione um ícone de checkbox clicável.
  2. Ao clicar no checkbox, chame uma função que usa `updateDoc` para alterar o campo `completed` da tarefa no Firestore.
  3. Use estilização condicional para que tarefas completas apareçam com um visual diferente (ex: riscadas).
  4. Adicione um ícone de lixeira em cada item da lista.
  5. Ao clicar na lixeira, chame uma função que usa `deleteDoc` para remover a tarefa do Firestore.
  6. Teste todo o fluxo: criar, ver, marcar como concluída e apagar uma tarefa.

Aula Prática 12: Refatorando com Zustand

Objetivo

Implementar uma 'store' com Zustand para gerir o estado de autenticação de forma centralizada e otimizada.

Passos

  1. Instale a biblioteca `zustand`.
  2. Crie uma pasta `src/store` e um ficheiro `authStore.js`.
  3. Crie uma 'store' que gira o estado do utilizador (`user`) e o estado de carregamento (`loading`).
  4. Mova a lógica do `onAuthStateChanged` para dentro da 'store', fazendo com que ela atualize o estado `user`.
  5. Refatore os componentes que dependem do estado de autenticação para consumir os dados da 'store' do Zustand.

Aula Prática 13: Adicionando Foto de Perfil

Objetivo

Permitir que o utilizador adicione uma foto de perfil usando a galeria do telemóvel com a API `expo-image-picker`.

Passos

  1. Crie uma tela `ProfileScreen.js` e adicione-a à navegação.
  2. Instale a biblioteca `expo-image-picker`.
  3. Na `ProfileScreen`, adicione um avatar padrão e um botão "Alterar Foto".
  4. Crie uma função que pede permissão para aceder à galeria.
  5. Se a permissão for concedida, abra a galeria com `launchImageLibraryAsync`.
  6. Quando o utilizador selecionar uma imagem, atualize o estado e exiba a imagem selecionada.
  7. **Bónus:** Pesquise sobre o Firebase Storage e implemente o upload da imagem para o backend.

Aula Prática 15: Teste Unitário e Build Final

Objetivo

Escrever um teste simples para garantir a qualidade do código e gerar o ficheiro de instalação da aplicação.

Passos

  1. Configure o Jest no projeto, se ainda não estiver.
  2. Escolha um componente simples e sem muita lógica (ex: o `Header` criado na aula 2).
  3. Escreva um teste que verifica se o componente renderiza o texto passado via `props` corretamente.
  4. Instale e configure o EAS CLI (`npm install -g eas-cli`).
  5. Faça login na sua conta Expo com `eas login`.
  6. Configure o projeto para o build com `eas build:configure`.
  7. Execute o comando de build para Android (`eas build -p android --profile preview`).
  8. Após a conclusão, descarregue o ficheiro APK e instale-o num dispositivo Android físico para ver o resultado final!
